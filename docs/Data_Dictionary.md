# Data Dictionary

This document outlines the schema for the raw sales data and the engineered features used in our analytical models.

## Raw Data (Source: `Cardenality_Sales_Data.xlsx`)

| Column Name | Data Type | Description |
| :--- | :--- | :--- |
| `OrderNumber` | String | Unique alphanumeric identifier for each customer transaction. |
| `Sales Channel` | String | The medium through which the sale was made (Values: `In-Store`, `Online`, `Distributor`). |
| `WarehouseCode` | String | Identifier for the warehouse facility fulfilling the order. |
| `ProcuredDate` | Date | The date the items were procured by the warehouse. |
| `OrderDate` | Date | The date the customer placed the order. |
| `ShipDate` | Date | The date the order left the warehouse. |
| `DeliveryDate` | Date | The date the customer received the order. |
| `CurrencyCode` | String | The currency used for the transaction (e.g., USD). |
| `_SalesTeamID` | Integer | Foreign key linking to the Sales Team dimension table. |
| `_CustomerID` | Integer | Foreign key linking to the Customer dimension table. |
| `_StoreID` | Integer | Foreign key linking to the Store dimension table. |
| `_ProductID` | Integer | Foreign key linking to the Product dimension table. |
| `Order Quantity` | Integer | The number of units purchased in the transaction. |
| `Discount Applied` | Float | The monetary value of the discount applied to the total order. |
| `Unit Price` | Float | The selling price per single unit of the product. |
| `Unit Cost` | Float | The cost incurred by the company to produce/acquire a single unit. |

---

## Engineered Features (Created via Python/SQL)

The following fields were derived during the ETL phase to facilitate advanced analysis.

| Column Name | Data Type | Calculation Logic | Description |
| :--- | :--- | :--- | :--- |
| `TotalRevenue` | Float | `(Order Quantity * Unit Price) - Discount Applied` | The actual top-line revenue generated by the transaction after discounts. |
| `TotalCost` | Float | `Order Quantity * Unit Cost` | The total cost of goods sold (COGS) for the transaction. |
| `TotalProfit` | Float | `TotalRevenue - TotalCost` | The net profit generated from the transaction. |
| `ProfitMargin_Pct` | Float | `(TotalProfit / TotalRevenue) * 100` | The percentage of revenue that translates into profit. |
| `DeliveryDays` | Integer | `DeliveryDate - OrderDate` (in days) | The total lead time experienced by the customer from order to delivery. |
| `DiscountTier` | String | Case statement categorizing `Discount Applied` | Groups discounts into 'No Discount', 'Low Discount (1-50)', and 'High Discount (50+)'. |
